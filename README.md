# Логіка (2021) — Цифровізація підручника

Цей проект присвячений процесу оцифрування, структурування та перетворення підручника з логіки (`pidr_logika_2021.pdf`) на інтерактивну персональну базу знань для програми **Obsidian**.

---

## 🎯 Призначення проекту

Перетворення традиційного лінійного PDF-підручника на сучасну мережеву базу знань (так званий "другий мозок" або Zettelkasten). Проект автоматизує:
1. Вилучення чистого тексту з PDF без втрати змісту.
2. Сегментацію тексту на 39 логічних підрозділів (тем) за картою сторінок.
3. Очищення тексту від системного шуму (колонтитулів, повторюваних заголовків та номерів сторінок на межах стиків).
4. Генерацію структури папок, YAML метаданих та перехресних вікі-посилань (Wikilinks) для миттєвого імпорту в **Obsidian**.

---

## 📂 Структура проекту

*   `pidr_logika_2021.pdf` — оригінальний файл підручника з логіки.
*   `parse_pdf.py` — Python-скрипт для зчитування PDF та збереження тексту в один файл.
*   `logica.md` — повний згенерований текст підручника, розділений за сторінками (`## Сторінка X`).
*   `segment_logica.py` — Python-скрипт, який розбиває `logica.md` на 39 окремих JSON-сегментів відповідно до карти сторінок у `skill_for_logica.md`. Він також автоматично видаляє номери сторінок та дубльовані назви розділів/колонтитулів.
*   `logica_json_segments/` — папка з 39 структурованими JSON-файлами. Кожен файл містить три поля: `"назва розділу"`, `"назва сегменту"` та `"повний текст сегменту"`.
*   `generate_obsidian_vault.py` — Python-скрипт, який конвертує JSON-сегменти у формат Markdown нотаток та створює структуру папок для Obsidian.
*   `Логіка_Obsidian/` — готова база знань (Vault) для Obsidian:
    *   `Зміст.md` — головна інтерактивна карта змісту (MOC) з посиланнями на всі теми.
    *   **Папки `01_`–`10_`** — структуровані розділи книги, що містять відповідні `.md` файли конспектів.
*   `skill_for_logica.md` — інструкція та точна карта розподілу сторінок підручника.
*   `extract_to_graph.py` — Python-скрипт для екстракції графу знань (Concept, Rule, Fallacy, Example) та зв'язків за допомогою LLM-моделей та запису в Neo4j.
*   `validate_graph.py` — Python-скрипт для контролю якості графу знань, перевірки зв'язків та стану векторних ембедінгів у Neo4j.
*   `populate_embeddings.py` — Python-скрипт для налаштування векторних індексів та генерації векторів для описів сутностей через Ollama (`nomic-embed-text`).
*   `query_graphrag.py` — консольний GraphRAG оркестратор та асистент, який виконує векторний пошук, обходить підграф та відправляє контекст у локальну LLM.
*   `app.py` — веб-додаток на Streamlit з чат-інтерфейсом, який візуалізує вилучений контекст з бази знань (вузли, ваги подібності та зв'язки) та виводить відповідь від Gemma 4.
*   `.gitignore` — налаштування файлів, які ігноруються в Git.

## 💻 Як користуватися проектом

### 1. Використання готової бази в Obsidian

Папка `Логіка_Obsidian` повністю готова до роботи.
1. Завантажте та встановіть додаток [Obsidian](https://obsidian.md/).
2. Відкрийте Obsidian і виберіть **"Open folder as vault"** (Відкрити папку як сховище).
3. Вкажіть шлях до папки `Логіка_Obsidian` на вашому комп'ютері.
4. Відкрийте файл `Зміст.md` у корені сховища.
5. Ви можете переходити по темах за допомогою посилань `[[Назва теми]]` (утримуйте `Ctrl` для швидкого перегляду або натисніть для переходу).
6. Кожна нотатка містить навігаційну панель внизу (← Попередня тема | Зміст | Наступна тема →) для зручного послідовного читання.

### 2. Запуск конвеєра обробки (Pipeline) з нуля

Якщо ви хочете повторити весь процес обробки тексту самостійно, виконайте такі кроки:

1. **Встановіть залежності**:
   Для вилучення тексту з PDF потрібна бібліотека `pdfplumber`. Встановіть її через термінал:
   ```bash
   pip install pdfplumber
   ```

2. **Крок 1: Парсинг PDF у MD**:
   Запустіть скрипт для зчитування сторінок:
   ```bash
   python parse_pdf.py
   ```
   Це створить файл `logica.md`.

3. **Крок 2: Сегментація на JSON**:
   Розбийте загальний файл на окремі очищені теми:
   ```bash
   python segment_logica.py
   ```
   Скрипт створить папку `logica_json_segments/` із 39 файлами.

4. **Крок 3: Генерація сховища Obsidian**:
   Створіть структуру папок і взаємопов'язані Markdown файли:
   ```bash
   python generate_obsidian_vault.py
   ```
   Буде створено або оновлено папку `Логіка_Obsidian/`.

## 🛠️ Технічні особливості нотаток

Кожен файл Markdown (`.md`) у базі знань має наступний формат:

*   **YAML Frontmatter** (Метадані):
    ```yaml
    ---
    розділ: "Назва розділу книги"
    сегмент: "Назва конкретного підрозділу"
    номер: Порядковий номер теми (1-39)
    попередня: "[[Посилання на попередній сегмент]]"
    наступна: "[[Посилання на наступний сегмент]]"
    тип: конспект
    ---
    ```
*   **Хлібні крихти (Breadcrumbs)** вгорі сторінки для швидкого орієнтування.
*   **Очищений текст**: Текст не містить переривань сторінок та дублікатів колонтитулів на кшталт `9\n1.1. Поняття про мислення`.

## 🧠 Інтеграція GraphRAG (Neo4j + Ollama + Gemma 4)

Проект підтримує повноцінний локальний контур RAG (Retrieval-Augmented Generation) на основі згенерованого графу логічних взаємозв'язків з підручника.

### Схема роботи GraphRAG:
1. **Векторний пошук (Vector Search)**: Запит користувача трансформується у вектор через `nomic-embed-text` та зіставляється з векторними індексами описів сутностей у Neo4j.
2. **Екстракція підграфу (Graph Traversal)**: Вилучаються найближчі сутності та їхні взаємозв'язки 1-2 рівнів глибини.
3. **Генерація відповіді**: Структурований контекст інтегрується в промпт для локальної LLM `gemma4:e4b` (або іншої обраної моделі в Ollama), яка відповідає на запитання без галюцинацій та вигадок.

### Запуск та налаштування:

1. **Запустіть локальні сервіси**:
   * Переконайтеся, що запущено Neo4j (за замовчуванням `bolt://localhost:7687`) та Ollama з моделями `nomic-embed-text` і `gemma4:e4b`.
   * Налаштуйте доступ у файлі `.env`.

2. **Заповніть векторні індекси**:
   Щоб система могла шукати сутності за сенсом, згенеруйте векторні ембедінги для описів сутностей та створіть індекси:
   ```bash
   python3 populate_embeddings.py
   ```

3. **Запустіть валідацію графу**:
   Перевірте кількість вузлів, зв'язків, глибину ієрархій та наявність ізольованих вузлів:
   ```bash
   python3 validate_graph.py
   ```

4. **Запуск у консольному режимі (CLI)**:
   Для тестування одного запиту або роботи в інтерактивному чаті:
   ```bash
   # Одиничний запит
   python3 query_graphrag.py --query "Суддя заявив, що оскільки підсудний запізнився, він точно винен. Яка тут помилка?"
   
   # Інтерактивний чат
   python3 query_graphrag.py --interactive
   ```

5. **Запуск інтерактивного веб-інтерфейсу (Streamlit UI)**:
   ```bash
   python3 -m streamlit run app.py
   ```
   Після запуску відкрийте [http://localhost:8501](http://localhost:8501) у браузері.
