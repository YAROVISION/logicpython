# Інструкція (Скіл) з сегментації книги "Логіка" у JSON

Цей документ містить детальні інструкції (промпт) для розбиття файлу `logica.md` на окремі змістовні сегменти (підрозділи) та їх збереження у форматі `.json`.

---

## Промпт (Інструкція для ШІ / Парсера)

Ви є спеціалізованим парсером тексту. Ваше завдання — розділити файл `logica.md` на логічні сегменти відповідно до наведеної нижче структури та зберегти кожен сегмент у окремий JSON-файл.

### Правила сегментації
1. **Визначення сторінок**: Орієнтуйтеся на розмітку `## Сторінка X` у файлі `logica.md`. Текст сторінки починається після заголовка `## Сторінка X` і закінчується перед наступним заголовком сторінки або розділювачем `---`.
2. **Межі сегментів**: Кожен сегмент охоплює певний діапазон сторінок (включно з початковою та кінцевою сторінками).
3. **Формат результату**: Кожен сегмент має бути збережений у форматі JSON з наступною структурою (3 обов'язкових поля):
   ```json
   {
     "назва розділу": "Назва відповідного розділу книги",
     "назва сегменту": "Назва конкретного підрозділу або теми",
     "повний текст сегменту": "Повний текст всіх сторінок, які належать до цього сегменту, очищений від технічних розділювачів сторінок (наприклад, '## Сторінка X' та '---')"
   }
   ```
4. **Очищення тексту**: При об'єднанні тексту сторінок для поля `"повний текст сегменту"` слід видалити рядки `## Сторінка X` та розділювачі `---`, але зберегти весь інший вміст, включаючи номери сторінок у самому тексті для забезпечення цілісності.
